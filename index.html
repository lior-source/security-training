2 Shift Left methodology & OWASP Top 10 for .NET
09:30 – 11:30

Shift Left means moving security testing earlier in the development lifecycle — instead of finding vulnerabilities after deployment (when they cost 100x more to fix), we catch them while the code is being written. This is the single most important concept in modern secure development.

Finding a bug during a code review takes minutes to fix. Finding the same bug in production — after a breach — takes weeks and costs tens of thousands of dollars. The goal of this training is to move your security mindset to the left: catch it while you code, not after it's deployed.

The OWASP Top 10 is the industry-standard list of the most critical web application security risks. Below is how each one applies to your .NET / C# codebase. Your task is to read the full OWASP documentation and connect each category to your own code.

A01Broken Access Control— missing [Authorize]
A02Cryptographic Failures— plaintext in appsettings
A03Injection— SQL via string concat
A04Insecure Design— no input validation
A05Security Misconfiguration— debug in production
A06Vulnerable Components— outdated NuGet pkgs
A07Auth Failures— weak passwords, no MFA
A08Data Integrity Failures— BinaryFormatter!
A09Logging Failures— logging PII data
A10SSRF— unvalidated HttpClient URLs

3 Secure coding patterns in C# — with code examples
11:30 – 12:00

Now that you understand the OWASP categories, let's look at how they show up in actual C# code. Below are common patterns — the dangerous version on the left, the secure version on the right. These are real patterns that appear in production .NET codebases every day.

DANGEROUS — SQL Injection
var query = $"SELECT * FROM Users
  WHERE Name = '{input}'";
// Attacker injects: ' OR 1=1 --
// Returns ALL users from the DB
SAFE — Parameterized Query
var cmd = new SqlCommand(
  "SELECT * FROM Users WHERE Name = @n");
cmd.Parameters.AddWithValue("@n", input);
// Input treated as data, never code
DANGEROUS — Hardcoded Secrets
var connStr = "Server=db;
  Password=P@ssw0rd123!";
// Anyone with repo access sees this
// If pushed to GitHub — game over
SAFE — User Secrets / Key Vault
var connStr = Configuration
  .GetConnectionString("Default");
// Stored in Azure Key Vault
// Rotated without code changes
DANGEROUS — Insecure Deserialization
var fmt = new BinaryFormatter();
var obj = fmt.Deserialize(stream);
// Remote Code Execution risk!
// BinaryFormatter is banned in .NET 8+
SAFE — System.Text.Json
var obj = JsonSerializer
  .Deserialize<MyType>(json);
// Type-safe, no code execution
// Modern .NET standard

5 The 4 security tools — understand what protects your code
14:30 – 15:30

Before we introduce our SecureStack pipeline, you need to understand each of the 4 tools that power it. These are all open-source, industry-standard tools used by companies like Google, Microsoft, and Netflix. Together, they cover every major category of code vulnerability.

Gitleaks
Secret Detection
Scans every commit for leaked passwords, API keys, tokens, and connection strings. Uses 150+ regex patterns. Scans your entire Git history — not just the latest code. If you ever accidentally committed a password, Gitleaks will find it.
Semgrep
Static Analysis (SAST)
Reads your C# code without running it. Finds insecure patterns like SQL injection, XSS, insecure deserialization, and missing auth attributes. Has 1,500+ security rules and understands code semantics — not just text matching.
Snyk
Dependency Scanning (SCA)
Checks every NuGet package in your project against CVE vulnerability databases. Finds transitive (indirect) vulnerabilities too — packages your packages depend on. Suggests exact upgrade versions to fix each issue.
Trivy
Container & IaC Scanning
Scans Docker images, Dockerfiles, and infrastructure-as-code files for OS-level vulnerabilities and misconfigurations. Even if your code is perfect, the base image might have 50+ vulnerabilities. Trivy catches those.
Your Task
  • Read the Gitleaks README on GitHub — understand what it detects and how to allowlist false positives (~10 min).
  • Browse the Semgrep C# rule library — find 3 rules relevant to your codebase (~15 min).
  • Run dotnet list package --vulnerable --include-transitive on your project — see what Snyk would find (~5 min).
  • Read the Trivy quickstart — understand container scanning basics (~10 min).
  • For each tool, write one sentence: "This tool would catch ____ in our codebase."

6 SecureStack — all 4 tools in one automated pipeline
15:30 – 16:00

Now you understand each tool individually. Here's how they work together. SecureStack is AlphaDevs' automated security pipeline that combines Gitleaks + Semgrep + Snyk + Trivy into a single system that runs on every Pull Request. You don't need to install anything or run any commands — when you push code and open a PR, all 4 scans trigger automatically.

If all checks pass — your PR can be merged. If any check fails — the PR is blocked until you fix the findings. No exceptions. This isn't about slowing you down — it's about catching issues in minutes instead of discovering them after a breach.
